상세한 설명

ECS 매니저의 중추 설계 헤더

엔티티를 등록하고, 컴포넌트를 등록하는 시스템이 정의됨.

타입정의 문서화

◆ Data

using MIR::ECS::Data
초기값:
std::unordered_map<Component::Tag,
std::unique_ptr<Component, std::function<void(Component*)>>>

엔티티가 보유한 컴포넌트들을 관리하는 컨테이너 타입

key: Component::Tag(컴포넌트 식별용), value: 커스텀 deleter가 적용된 컴포넌트 unique_ptr. 이를 통해 엔티티의 컴포넌트의 할당 동적 관리.

◆ Mask

using MIR::ECS::Mask = std::bitset<MAX_COMPONENTS>

엔티티의 컴포넌트 보유 상태를 나타내는 비트 마스크 타입

MAX_COMPONENTS 크기의 비트셋을 통해 엔티티가 어떤 컴포넌트를 가지고 있는지 알 수 있음. 각 비트는 특정 컴포넌트 유형을 나타내며, 비트가 1이면 해당 컴포넌트를 보유한 것을 의미.

◆ Pool

using MIR::ECS::Pool = std::shared_ptr<PoolBase>

컴포넌트 풀(Pool)을 가리키는 스마트 포인터 타입

풀을 shared_ptr로 관리하여 다형성 유지 가능.
